# Поради для покращення PageSpeed

Що вже зроблено на сайті:
- Gzip (compression) на сервері
- Кешування статики (1 день у production)
- **Мініфікація CSS і JS** у production: `npm run build` створює `styles.min.css` і `main.min.js`; сервер при `NODE_ENV=production` віддає їх замість повних файлів.
- Preload основного CSS
- Lazy loading для зображень нижче екрану (відгуки, картки товарів)
- Defer для основного JS
- fetchpriority="high" для hero-зображення
- Preconnect для Google Fonts
- width/height на зображеннях карток товарів (менший CLS)

---

## Що ще можна зробити

### 1. Зображення (найбільший ефект)
- **Стиснути** всі JPG/PNG (TinyPNG, ImageOptim, Squoosh).
- **WebP** — конвертувати фото товарів і hero у WebP; у HTML/JS підставляти `.webp` або використовувати `<picture>` з fallback на JPG/PNG.
- **Розміри** — зберігати та віддавати зображення в потрібному розмірі (наприклад, мініатюри каталогу окремо від великих фото для модалки), щоб не завантажувати зайві кілобайти.

### 2. Шрифти
- Зараз використовується Google Fonts з `display=swap` — це вже добре.
- Можна **самохостити** шрифт Outfit (скачати .woff2 і підключити через `@font-face`), щоб прибрати залежність від fonts.googleapis.com і скоротити запити.

### 3. CSS і JS
- **Мініфікація** — перед деплоєм мініфікувати `css/styles.css` і `js/main.js` (наприклад, через Vite, Parcel або онлайн-мініфайери). Це зменшить розмір файлів і трохи пришвидшить завантаження.
- **Критичний CSS** — винести стилі для першого екрану (header, hero) в inline `<style>` у `<head>`, решту підвантажувати асинхронно або як звичайний файл. Це зменшує блокуючу роботу CSS на початку завантаження.

### 4. Сервер і хостинг
- На **Render** переконатися, що увімкнено **HTTPS**.
- Якщо хостинг підтримує **Brotli**, увімкнути його (часто дає краще стиснення ніж gzip).
- Для статики (CSS, JS, картинки) залишати кеш на 1–7 днів; для `index.html` можна коротший кеш (наприклад, 1 година) або без кешу, якщо часто оновлюєте контент.

### 5. Інше
- **Мета-теги** — додати `description` і `keywords` для SEO; на PageSpeed це не дуже впливає, але корисно для пошуку.
- **Розміри зображень у HTML** — де можливо, вказувати `width` і `height` у пікселях, щоб браузер зарезервував місце і не було стрибків макету (кращий CLS).

Після змін варто знову перевірити сторінку в [PageSpeed Insights](https://pagespeed.web.dev/) (Mobile і Desktop) і дивитися розділи «Diagnostics» та «Opportunities» — там будуть конкретні поради для вашого URL.
